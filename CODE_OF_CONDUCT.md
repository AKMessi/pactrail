# Code of conduct

Pactrail is committed to a respectful, inclusive, technically rigorous
community. Participate in a way that makes collaboration safe for people of all
backgrounds and experience levels.

## Expected behavior

- Be respectful, specific, and constructive.
- Critique ideas and implementations rather than people.
- Assume good intent while accepting correction and responsibility for impact.
- Welcome questions and explain decisions without gatekeeping.
- Respect privacy, attribution, licenses, disclosure embargoes, and project data
  rules.

## Unacceptable behavior

Harassment, discrimination, threats, sexualized attention, doxxing, stalking,
deliberate intimidation, sustained disruption, bad-faith manipulation, or
publishing another person's private information is not tolerated. Neither is
using security reports or private project data to pressure contributors.

## Scope and enforcement

This policy applies in project repositories, reviews, discussions, releases,
community spaces, and when representing Pactrail publicly. Maintainers may edit
or remove content, issue a warning, temporarily restrict participation, or ban a
participant based on severity and pattern.

Report conduct concerns privately to the repository owner through the contact
options on the [AKMessi GitHub profile](https://github.com/AKMessi). Reports will
be handled as confidentially as practical. Maintainers involved in a report must
recuse themselves from deciding it.
