# Contributing to Pactrail

Pactrail welcomes focused issues, design discussions, tests, documentation, and code contributions.

1. Discuss material protocol, security-boundary, or architecture changes in an issue before implementation.
2. Keep commits reviewable and add tests for observable behavior.
3. Run formatting, Clippy, and the complete workspace test suite.
4. Add a Developer Certificate of Origin sign-off to commits: `git commit -s`.
5. Do not include credentials, proprietary source code, model transcripts, or generated artifacts in reports.

By contributing, you certify the contribution under the [Developer Certificate of Origin 1.1](https://developercertificate.org/).
